```https://github.com/ubiquity/pay.ubq.fi/issues/73
This [payout page](https://pay.ubq.fi/) is responsible for bounty claims ([claim example](https://github.com/ubiquity/pay.ubq.fi/issues/52#issuecomment-1524678816)). Bounty claims are generated with the help of the uniswap's [permit2 contract](https://github.com/Uniswap/permit2). Each issue has a unique bounty claim URL thanks to the [permit2 nonce](https://github.com/Uniswap/permit2/blob/main/src/SignatureTransfer.sol#L19).

Sometimes there are cases when we should invalidate [nonces](https://github.com/Uniswap/permit2/blob/main/src/SignatureTransfer.sol#L19) for some claim URLs in case of the bot's error or smth else. That is why we need a friendly UI for nonce invalidation.

What should be done:
1. Add a new button "invalidate" to the https://pay.ubq.fi page
2. The "invalidate" button should be visible only if the authorized wallet account equals to the permit signer and the nonce is unclaimed
3. On "invalidate" button click we should send a tx that invalidates the nonce

On-chain example of nonce invalidation [here](https://github.com/ubiquity/pay.ubq.fi/blob/main/test/Permit2.t.sol#L333)
```
